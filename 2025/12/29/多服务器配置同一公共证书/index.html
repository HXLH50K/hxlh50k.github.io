多服务器配置同一公共证书

发表于2025-12-29更新于2025-12-29

本文记录了在多台服务器上使用 acme.sh 和 DNS API 为同一个泛域名申请和续订 SSL 证书的过程。

准备条件

  • 一个域名, 例如 abc.org
  • 一台以上的服务器
  • 每一台服务器最终会有单独的子域名, 例如 us1.abc.org, jp2.abc.org

1. 获取域名解析商 API 凭证

这一步根据域名解析商而异。本文以 Microsoft Azure 为例。

  1. 登录 Azure Cloud Shell。
  2. 获取订阅 ID：
  3. 创建一个服务主体 (Service Principal) 并获取其凭证。将 <SubscriptionID> 替换为上一步中获取到的 ID。
  4. 将获得类似以下的 JSON 输出，请妥善保管 appId, password, 和 tenant 的值。

2. 在服务器上申请证书

在每一台需要配置证书的服务器上执行以下步骤。

  1. 切换到 root 用户。使用 sudo 可能会在后续更新证书时遇到权限问题。
  2. 将上一步获取的凭证设置为环境变量。

    说明: acme.sh 在首次成功使用 DNS API 后，会将凭证加密保存在 ~/.acme.sh/account.conf 中。因此，这些环境变量仅在首次申请证书时需要，后续续订任务会自动加载已保存的凭证。

  3. 安装 acme.sh。

    安装脚本会将 acme.sh 的路径添加到 .bashrc 文件中。执行 source ~/.bashrc 可使其在当前终端立即生效。为确保命令在所有新终端中都可用，建议重新登录或开启一个新的终端窗口。

  4. 将默认 CA 设置为 Let’s Encrypt。

3. 配置服务与 DNS

以 Hysteria2 为例。

  1. 安装证书
    创建目录并将 acme.sh 申请到的证书安装到指定位置。--reloadcmd 会在证书自动续订后重启相应服务。

    *此处的systemctl restart hysteria2中的systemd 服务名称是在上一篇文章中配置的, 具体取决于你本地设置。

  2. 配置服务
    修改服务配置文件（例如 Hysteria2 的 config.yaml），将证书路径指向 /etc/hysteria/cert.crt 和 /etc/hysteria/private.key，然后重启服务。

  3. 配置 DNS 解析
    在域名托管商的 DNS 设置页面，为服务器 IP 添加一条 A 记录，指向分配给这台服务器的子域名。

    类型 名称 (子域名) 内容 (服务器 IP)
    A us1 123.123.123.123

4. 客户端连接

完成以上步骤后，修改客户端配置：

  • 服务器地址: us1.abc.org (或你设置的子域名)
  • SNI/服务器名称: us1.abc.org (通常与服务器地址一致，客户端会自动填充)
  • 跳过证书验证 (Allow Insecure): false

此时客户端应当可以正常连接。

总结

对于每一台新的服务器，只需重复执行【2. 在服务器上申请证书】和【3. 安装证书并配置服务】的步骤即可。acme.sh 会自动处理证书的续订事宜。
